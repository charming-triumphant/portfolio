## Phase 2: Pattern Recognition — Six Thinking Hats

### ⬜ White Hat (Facts)
- Hiring managers spend ~30 seconds before deciding to stay or leave
- Mobile-first — they WILL Google you on their phone
- NZ tech has two clusters: Wellington (gov/fintech/reliability) and Auckland (startups/speed)
- 3D with Three.js/R3F has real learning curve and performance implications
- `.dev` domain costs ~$12/year — no reason not to
- Google takes 2–6 weeks to index after sitemap submission
- The average junior portfolio sets an extremely low bar

### 🔴 Red Hat (Emotions)
- 🔥 **Feels immediately right:** 3D chair-destroyer hero, personality micro-copy, console.log easter egg, above-fold contract, proof > claims
- ⚠️ **Feels risky:** ambient sound (could annoy), physics interactions if unpolished
- 💚 **Feels underrated:** learning velocity signal, "last updated" date, CV matching portfolio design

### 🟡 Yellow Hat (Benefits — Tier 1 Must-Build)
1. Portfolio as own case study
2. 3D interactive hero (minimalist)
3. Proof > Claims everywhere
4. Above-the-fold contract
5. yourname.dev domain
6. Quantify everything
7. 3 projects, deep case studies
8. Sitemap + OG images + schema markup

### ⚫ Black Hat (Risks)
- **Complexity trap:** Don't do ALL of 3D + physics + cursor + sound — pick one headline technical flex
- **Performance paradox:** Three.js can destroy Lighthouse if unoptimised — performance work is mandatory if going 3D
- **Maintenance debt:** "Right Now" + GitHub live feed only if you'll actually update them
- **Try-hard zone:** Too many easter eggs/jokes = trying too hard; one wow interaction, rest is polish
- **Reasoned stack backfire:** Only claim reasoning you can defend in an interview

### 🟢 Green Hat (Synthesis Ideas)
- **[Synthesis A]** Evidence Trail Architecture — whole site structured as a court case
- **[Synthesis B]** One Project, Three Depths — surface, design, and code layers in one hero case study
- **[Synthesis C]** Performance + 3D Paradox as Feature — "I built 3D WebGL and still scored 95+ Lighthouse, here's how"
- **[Synthesis D]** Minimalist 3D — ONE perfect 3D moment in the hero, clean minimalism everywhere else ✅ **CHOSEN**

### 🔵 Blue Hat (Priorities)
**Non-negotiables:** yourname.dev, minimalist 3D hero, portfolio-as-case-study, 3 deep case studies, proof>claims, above-fold contract, sitemap/OG/schema, console.log easter egg
**High-value additions:** custom cursor, scroll animations, learning velocity, anti-skills-list, right-now section, testimonial, distinctive palette, 404 personality
**Cut:** ambient sound (risky for corporate), multiple 3D scenes per project, recruiter/developer mode toggle

## Phase 4: Action Planning — Solution Matrix & Build Roadmap

### Contact & Connect Design Principles
- **3 entry points:** Hero CTA (scroll to contact) + Mid-page availability block + Dedicated contact section
- **Form:** Name, Email, Message ONLY — nothing else
- **Visible email address** in plain text — don't force form use
- **Response promise:** "I reply within 24 hours"
- **Inline confirmation** — no redirect, personalised "sent to [email]" message
- **Download CV button** in: nav bar, mid-page availability block, contact section
- **CV filename:** `Ben-[Lastname]-CV-2026.pdf` — professional, not `resume_final_v3.pdf`
